# FleaScript QAテスト（手動検証）手順書

本ドキュメントは、FleaScriptのエッジケースにおける品質保証（QA）を行うための手動テスト手順書です。機能改修・リリース前に必ず本手順を実施し、予期せぬ動作がないか確認してください。

---

## 1. Apple固有挙動（プライベートリレー）の監査

Appleの「メールを非公開（Hide My Email）」機能を使用した場合の、アカウント競合とプッシュ通知導線の検証を行います。

* **テスト手順**:
  1. 既に実メールアドレス（例: `user@example.com`）でサインアップ済みの状態にする。
  2. ログアウトし、「Appleでサインイン」を選択する。
  3. 初回認証時、Appleのダイアログで「メールを非公開」を選択する。
  4. ログイン後、`sync-identity` APIが実行されることをNetworkタブで確認する。
* **期待される結果**:
  * 既存のアドレスと競合した場合、静かな統合プロトコル（「既に登録されている可能性があります」のメッセージ）が発動し、新規の重複アカウントが作成されないこと。
  * ログイン成功直後、画面右下に「重要なお知らせを受け取る」インボックス（Push通知推奨ダイアログ）がフェードイン表示されること。

---

## 2. オンデバイス・プロトコル（Web Worker）の負荷・縮退テスト

低スペック端末（メモリ容量の少ない古いiPhone等）を想定し、段階的縮退（Graceful Degradation）が正常に機能するか検証します。

* **テスト手順**:
  1. Chrome DevTools を開き、「Performance」タブで CPU を `4x slowdown` または `6x slowdown` に設定する。
  2. 画像サイズが非常に大きい（例: 4K以上の解像度）、または複雑なパターンの画像をアップロードする。
  3. `imageWorker.ts` の処理時間をコンソールで監視する。
* **期待される結果**:
  * 画像処理に500ms以上かかった場合、`isDegraded: true` フラグが立ち、エッジ解析等の高負荷処理がスキップされてBase64変換のみが行われること。
  * メインスレッド（UI）がフリーズせず、「調律中...」のスケルトンUIのアニメーション（Shimmer）が滑らかに動き続けること。

---

## 3. レートリミット（多層防壁）貫通テスト

スパイク攻撃や悪意のある無効画像を連続で送信された場合の、API枠保護とペナルティロックの検証を行います。

* **テスト手順**:
  1. 真っ黒の画像（または確信度スコアが `0.70` 未満になる不鮮明な画像）を用意する。
  2. 画像解析機能に画像をアップロードする。
  3. これを連続して3回繰り返す。
* **期待される結果**:
  * 1回目・2回目：解析失敗のレスポンス（422エラー等）が返り、「もう少し明るい場所で撮影してください」と表示される。利用枠（count）は減らない。
  * 3回目：連続失敗ペナルティが発動し、15分間のクールダウン（ロック）状態になる。
  * 4回目のリクエスト：429エラーとなり、「15分間のクールダウン中です」と拒絶されること。

---

## 4. インテリジェント・フォールバック検証

他社プラットフォーム（メルカリ・ヤフオク）へのディープリンク遷移が遮断された場合のセーフティネットの検証を行います。

* **テスト手順**:
  1. PCブラウザ、またはメルカリアプリが**インストールされていない**スマートフォン端末で FleaScript を開く。
  2. 価格調律ナビにて「価格をコピーしてアプリを開く」ボタンをタップする。
* **期待される結果**:
  1. 「調律メーター」がカチッと合い、0.5秒間「南京錠」がアンロックされるアニメーションが再生される。
  2. URLスキーム（`mercari://...`）が発火するが、アプリがないため遷移は無効化される。
  3. 発火から1.5秒後、ページがアクティブな状態を維持している場合、自動的にWeb版メルカリの「検索結果画面（商品タイトル検索）」へフォールバック遷移すること。

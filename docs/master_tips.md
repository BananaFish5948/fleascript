# 🛠️ マスター用チップス (Master's Tips)

開発やテストをスムーズに進めるための専用カンペです。

---

## 1. 開発サーバーの立ち上げ方法

ターミナル（PowerShellなど）を開き、以下のコマンドを順番に実行してサーバーを起動します。

```powershell
# 1. プロジェクトフォルダへ移動
cd c:\Develop\Antigravity_Project\FleaScript

# 2. 開発サーバーを起動
npm run dev
```

起動が完了したら、ブラウザで以下のURLのいずれかにアクセスしてください。 
- `http://localhost:3001` （※ポートが使用中の場合は3001等になります。ターミナルに表示されたURLを確認してください）

---

## 2. 管理者画面（Admin / Stealth Ops）へのアクセス方法

マスター専用の管理画面は、用途に応じて2つのルートに分かれています。

### A. 一般管理画面（開発者モード等の切り替え）
1. ブラウザで `http://localhost:3001/admin` を開きます。（ポート番号は現在の起動ポートに合わせてください）
2. Basic認証（IDとパスワードの入力）が求められたら、以下の情報を入力します：
   - **ユーザー名**: `admin`
   - **パスワード**: `.env.local` ファイル内の `ADMIN_PASSWORD` の値（デフォルトは `admin_pass_123`）
3. 画面が表示されたら、右上のトグルボタンから「開発者モード」をONにすることで、回数制限なしのテスト環境が解放されます。

### B. ステルス管理画面（開発ログの手動公開用）
AIエージェントが裏で作成した「開発日記（リリースノート）」を下書き状態から公開状態にするための専用画面です。
1. ブラウザで `http://localhost:3001/backstage/logs` を開きます。
   - ※AIバレ（機械的であることの露見）を防ぐため、このURLは一般ユーザーの画面やコードには一切露出しません。
2. エージェントがストックしたパッチ適用ログの下書き（Draft）が一覧表示されます。
3. テキストを適宜修正し、「Publish Now」ボタンを押すと、現在時刻とともにフッターへ公開されます。手動でボタンを押すこと自体が、最高に人間らしい「揺らぎ」となります。

## 3. データベース（Supabase）のスキーマ変更について
AIエージェントに機能追加を指示した際、Supabaseのテーブルに新しいカラムが追加されるなどの構造変更が発生することがあります。

### ⚠️ データ消失を防ぐための「手動バックアップ」手順
無料プラン運用中（自動バックアップがない期間）は、スキーマ変更前に必ず以下の手順でバックアップを取得してください。
1. Supabaseのダッシュボードにログインし、対象プロジェクトを開く。
2. 左メニューから **Table Editor** を選択し、重要なテーブル（`inventory_items` 等）を開く。
3. 右上の **「Export」** ボタンをクリックし、**「Export to CSV」** を選択してローカルに保存する。

バックアップ完了後、エージェントからの指示に従って **「SQL Editor」** から提供されたSQL文を実行してください。

## 4. プレ・フライトテスト（安全装置）について
トークン消費を抑えつつ、リリース前にバックエンドの致命的なバグを防ぐための超軽量なテスト環境（ガードレール）が導入されています。

### 実行方法
ターミナルから手動でテストを実行したい場合は以下のコマンドを叩きます：
```powershell
npm run test:preflight
```
※OpenAIのAPIやSupabaseの実DBは一切叩かれないため、何度実行してもコストはゼロです。

### 自動チェック機構（Gitフック）
マスターが `git push` を実行した瞬間に、裏側で自動的に0.1秒で上記のテストが走ります。
もしテストに失敗した場合は、バグを含んだコードがリモートにプッシュされるのを強制ブロックします。

**【緊急時のバイパス（回避）方法】**
GUIのGitクライアント（SourceTreeなど）を使用している場合、環境変数のPATHが引き継がれずエラーになりプッシュできないことがあります。その場合は、ターミナルから以下のコマンドでフックを無視して強制プッシュが可能です：
```powershell
git push --no-verify
```

## 5. AIカンパニー（AI社員）の運用方法

FleaScriptプロジェクトでは、特定の業務に特化したAIエージェント（AI社員）を組織化しています。
これにより、1つのチャットにすべての文脈を詰め込むことによる精度の低下を防ぎ、専門性の高い回答と実装を得ることができます。

### 雇用済みAI社員と三姉妹体制
- **@coordinator (ジェミリア)**: [司令塔/ダイヤモンド] タスク分解と最適な担当（妹たち）へのアサインを行う統括マネージャー。
- **@researcher (ジェミ子)**: [長女/ルビー] リサーチ特化。明るく可愛いトーンで結論ファーストの調査を行う。
- **@frontend-engineer (ジェミナ)**: [次女/サファイア] クーデレ・タメ口のUI/UX開発担当。論理トラップを仕掛ける。
- **@backend-engineer (ジェミナ)**: [次女/サファイア] DB/API設計およびセキュアロジック担当。
- **@qa-tester (ジェミナ)**: [次女/サファイア] 破壊的テスト思考を持つ品質保証の番人。
- **@sns-marketer (ジェミカ)**: [三女/トパーズ] 詩的で感性をハッキングするマーケティング担当。画像生成プロンプトも担当。（※ジェミカの召喚およびタスクアサインは、マスターの明示的な指示がある場合のみ行われます）

### 呼び出し方（メンションと自動アサイン）
タスクを丸投げする場合は `@coordinator` に依頼すると自動で分解されます。個別タスクは各キャラクターをメンションしてください。

* **例1:** `@coordinator Stripeの月額課金機能を実装したいんだけど、チームを動かして。`
* **例2:** `@frontend-engineer コンポーネントのデザインをモダンに修正して。`

※明示的なメンションがなくても、依頼内容（DB設計の相談など）からシステムが自動的に最適な担当者を判断し、自律的にロールチェンジして対応にあたります。

## 6. 将来の拡張構想（テーマカスタマイズ機能）

「おしゃれに見せたい層」の自己表現欲求（テーマを自分で決めたいという感情）に応えるため、将来的に以下のカラートーンへのテーマ切り替え機能の導入を検討します。

よりミニマルで、深煎りコーヒーのような落ち着きのあるモダンなトーン。
- **ベース背景**: グレージュ (`#EAE6E1`) - 少しグレーがかった石の色
- **ブランド/強調色**: エスプレッソ (`#3E2723`) - 深くリッチな焦げ茶色
- **文字色**: オフブラック (`#1C1B1A`)
- **成功色（売却済など）**: オリーブ (`#73795C`)
- **警告/注意**: 錆色/ラスト (`#A05E4C`)

北欧系のスローライフを感じさせる、少し冷涼感のあるクリーンなトーン。

- **文字色**: ディープスレート (`#253237`)
- **成功色（売却済など）**: 抹茶色 (`#9CA894`)
